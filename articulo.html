Modelo OSI

Capa Física (Physical Layer)

La Capa Física es la capa más baja del modelo OSI, encargada de la transmisión y recepción de señales eléctricas, ópticas o electromagnéticas a través del medio de comunicación físico, como cables de cobre, fibra óptica o ondas de radio. Su función principal es convertir los bits de datos en señales físicas y viceversa, asegurando una transferencia confiable de información entre dispositivos conectados

Protocolos

Bluetooth: Es un estándar de comunicación inalámbrica de corto alcance utilizado para la transferencia de datos entre dispositivos, como teléfonos móviles, computadoras y periféricos.

Fibra Óptica: Utiliza cables de fibra óptica para transmitir datos a través de pulsos de luz. Ofrece una alta velocidad de transmisión y una mayor capacidad de ancho de banda en comparación con los cables de cobre.

RS-232 (Recommended Standard 232): Es un estándar de interfaz para la transmisión serial de datos entre dispositivos. Aunque es más antiguo, todavía se utiliza en algunas aplicaciones, como la conexión de periféricos a computadoras.



Capa de Enlace de Datos (Data Link Layer)

La Capa de Enlace de Datos es la segunda capa del modelo OSI que se encarga de la transferencia fiable de datos entre nodos conectados directamente. Esta capa proporciona la detección y corrección de errores, control de flujo y control de acceso al medio. Además, se encarga de dividir los datos en tramas para su transmisión y de reensamblarlas en el destino.

Protocolos

Asynchronous Transfer Mode (ATM): Un protocolo de conmutación de paquetes que utiliza celdas fijas de 53 bytes para la transmisión de datos. Se utiliza comúnmente en redes de alta velocidad y servicios de telecomunicaciones.

High-Level Data Link Control (HDLC): Es un protocolo de enlace de datos síncrono que se utiliza para la comunicación punto a punto y multipunto. Es comúnmente utilizado en redes WAN y es el protocolo predeterminado en muchas conexiones seriales.

Point-to-Point Protocol (PPP): Un protocolo comúnmente utilizado para establecer una conexión directa entre dos nodos en una red, como la conexión de un computadora a través de línea telefónica a un proveedor de servicios de Internet (ISP).



Capa de Red (Network Layer)

La Capa de Red es la tercera capa del modelo OSI, responsable de facilitar la comunicación entre dispositivos que pueden estar ubicados en redes diferentes. Su función principal es el enrutamiento, que implica determinar la mejor ruta para la transferencia de datos desde el origen hasta el destino a través de la red, teniendo en cuenta factores como la topología de la red, la congestión y la disponibilidad de rutas.

Protocolos

ARP (Address Resolution Protocol): Se utiliza para mapear direcciones de capa de red (IP) a direcciones de capa de enlace (como direcciones MAC en redes Ethernet). Es crucial para establecer la correspondencia entre direcciones lógicas y físicas.

IPsec (Internet Protocol Security): Un conjunto de protocolos de seguridad utilizado para asegurar las comunicaciones en la Capa de Red. IPsec proporciona autenticación, integridad, confidencialidad y otros servicios de seguridad para el tráfico IP.

IP (Internet Protocol): Protocolo fundamental de la Capa de Red en la suite TCP/IP. IP se encarga de la asignación de direcciones lógicas (direcciones IP) y del enrutamiento de paquetes entre redes.



Capa de Transporte (Transport Layer)

La Capa de Transporte es la cuarta capa del modelo OSI y se encarga de la transferencia confiable y eficiente de datos entre dispositivos finales en una red. Proporciona servicios de extremo a extremo, asegurando que los datos se entreguen de manera ordenada, sin errores y controlando la congestión en caso necesario. Los protocolos de transporte más comunes en esta capa son TCP (Transmission Control Protocol) y UDP (User Datagram Protocol).

Protocolos

DCCP (Datagram Congestion Control Protocol): Un protocolo de transporte experimental diseñado para aplicaciones que requieren comunicación de datagramas con control de congestión, similar a TCP, pero con un enfoque más flexible.

UDP (User Datagram Protocol): Un protocolo de transporte sin conexión que ofrece una transmisión más rápida pero menos confiable que TCP. Se utiliza en aplicaciones donde una entrega inmediata es más crítica que la integridad de todos los datos, como transmisiones en tiempo real y juegos en línea.

QUIC (Quick UDP Internet Connections): Un protocolo de transporte experimental desarrollado por Google que opera sobre UDP. QUIC está diseñado para mejorar la velocidad de carga de páginas web y otras aplicaciones mediante la reducción de la latencia y la optimización del rendimiento en conexiones inestables.



Capa de Sesión (Session Layer)

La Capa de Sesión es la quinta capa del modelo OSI, encargada de establecer, gestionar y cerrar sesiones entre aplicaciones en dispositivos de red. Su función es proporcionar un diálogo estructurado entre las aplicaciones, permitiendo la comunicación y la transferencia de datos de manera coherente y organizada. La Capa de Sesión también maneja la sincronización y el control de flujo entre las aplicaciones.

Protocolos

SSH (Secure Shell): Aunque comúnmente asociado con la Capa de Aplicación, SSH puede tener funciones de sesión al proporcionar un entorno seguro para la comunicación entre dispositivos. Facilita la autenticación y el cifrado de la comunicación.

AFP (Apple Filing Protocol): Protocolo utilizado para compartir archivos y servicios en entornos Macintosh. Incluye funciones que gestionan sesiones de usuario y conexión.

SMB (Server Message Block): Utilizado para compartir archivos, impresoras y otros recursos en una red. Puede involucrar la gestión de sesiones y conexiones.



Capa de Presentación (Presentation Layer)

La Capa de Presentación es la sexta capa del modelo OSI y tiene la responsabilidad de la representación de datos, la traducción y el cifrado. Su función es garantizar que los datos enviados desde la capa de aplicación en un extremo de la comunicación sean comprensibles para la capa de aplicación en el otro extremo, independientemente de las diferencias en la representación de datos entre los sistemas. También se ocupa de la compresión y el cifrado de datos para garantizar la seguridad y la eficiencia en la transmisión

Protocolos

HTML (Hypertext Markup Language): Aunque HTML se asocia principalmente con la Capa de Aplicación, también incluye elementos relacionados con la presentación de datos en la web, como la estructuración y el formato de texto e imágenes.

GIF (Graphics Interchange Format): Un formato de archivo de imagen que utiliza la compresión de datos sin pérdida, comúnmente utilizado para representar gráficos simples y animaciones en la web.

MIDI (Musical Instrument Digital Interface): Un protocolo estándar para la comunicación entre dispositivos electrónicos musicales y software de música. La Capa de Presentación podría participar en la representación de datos relacionados con la música.



Capa de Aplicación (Application Layer)

La Capa de Aplicación es la séptima y última capa del modelo OSI, y representa la interfaz entre el usuario y la red. En esta capa, se encuentran las aplicaciones de software que permiten a los usuarios realizar tareas específicas, como navegar por la web, enviar correos electrónicos, transferir archivos, entre otras. También incluye servicios de red y funciones como la autenticación, la autorización y la gestión de sesiones.

Protocolos

DNS (Domain Name System): Resuelve nombres de dominio legibles por humanos en direcciones IP, facilitando la navegación en la web al asociar nombres de dominio con direcciones IP correspondientes.

SSH (Secure Shell): Proporciona un método seguro para acceder a servidores remotos y ejecutar comandos de manera segura. También se utiliza para la transferencia segura de archivos (SFTP).

POP3 (Post Office Protocol version 3): Protocolo utilizado para la recuperación de correos electrónicos desde un servidor de correo. Permite a los clientes de correo electrónico descargar mensajes desde el servidor.
